# 🔎 문제 인식

> 자사 분석과 경쟁사 분석을 통해 현재 ‘스마트커버 인슈어런스’ 사이트의 문제를 파악했습니다.
> 

## **❇️ 자사 분석**

‘스마트커버 인슈어런스’의 시장 현황과 SWOT을 분석했습니다.

**1️⃣ 자사 현황**

- ‘스마트커버 인슈어런스’는 **반려동물 헬스케어 분야 선두 기업** ‘핏펫’의 자회사 법인보험대리점입니다.
    - 핏펫은 2017년 설립되어 모바일 반려동물 검진키트 ‘어헤드’, AI 서비스 기반 반려동물 물품 이커머스 ‘핏펫몰’, 반려동물 맞춤 건강관리 프로그램 ‘핏펫 헬스’, 동물병원 리뷰 및 예약 서비스 ‘핏펫 케어’ 등의 주요 사업을 추진 중임
    - 국내 반려동물 헬스 케어 분야 1위
- ‘스마트커버 인슈어런스’는 **펫보험 비교 서비스**를 통해 **펫 전문 보험사** 설립 목표를 지녔습니다.
    - 모회사 ‘핏펫’의 다양한 반려동물 사업으로 누적된 빅데이터를 활용해 2024년 2월 출범
    - 반려동물 빅데이터를 활용해 펫 전문 보험사 설립 추진
    - 반려동물 보호자의 상담과 컨설팅을 통해 펫보험을 추천
    - 주요 5개 손해보험사와 제휴를 맺고 보험 상품을 판매
    - 24년 8월 기준 약 2천여건의 펫보험 고객 상담 및 100건 이상의 펫보험 체결
    

**2️⃣ SWOT 분석**

- SWOT 분석 전략 중 **WO 전략**을 채택하여 **약점을 보완**하고
스마트커버인슈어런스가 타사에 비해 가지는 **강점과 기회 요인을 강화**하는 방향성을 구축했습니다.

보험사 출신 **전문가와의 상세한 상담**이라는 강점이 있지만

## ❇️ 경쟁사 분석

카카오페이, 비마이펫, 펫핀스 분석을 통해 ‘스마트커버인슈어런스’ 사이트의 보완점을 모색했습니다. 

**1️⃣ 웹사이트의 기능이 명확히 드러나는 슬로건** 

- 세 경쟁사 모두 **펫보험 ‘비교’ 서비스 기능**을 명확히 드러내는 슬로건을 내세워
사용자가 한눈에 **웹사이트의 정체성**을 파악할 수 있도록 유도했습니다.
    
    > *“다양한 보험사의 상품을 한눈에 **비교해요**-펫보험으로 병원비 부담을 낮출 수 있어요.”* (카카오페이)
    *”펫보험, 간단한 정보로 **바로 확인!**”* (비마이펫)
    *“AI **비교가입**과 청구까지 간편하게 펫보험은 펫핀스에서”* (펫핀스)
    > 

2️⃣ **사용자 피로도를 줄인 보험 정보 입력 과정** 

- 세 경쟁사 모두 **객관식 옵션 체크, 드롭다운, 스크롤 형식**을 통해
보험 정보 입력 과정을 단순화했습니다.
    
    > 객관식 체크 형식, 스크롤 (카카오페이, 비마이펫, 펫핀스)
    보장혜택 선택 가능 (비마이펫, 펫핀스)
    상품 클릭 시 세부 페이지 이동 (비마이펫)
    > 

**3️⃣ 차별적인 기능을 통한 사용자 경험 증진**

- 세 경쟁사 모두 **간편한 보험 비교 결과 안내**와 **부가적인 기능**으로 
사용자 경험을 증진했습니다.
    
    > 즉각적인 비교 결과 확인 가능 (카카오페이, 비마이펫, 펫핀스)
    갱신형/재가입형 분류 확인, 선택한 가입조건 수정 및 재확인 가능 (카카오페이)
    커뮤니티 Q&A 확인, 보험사 페이지 연결 (비마이펫)
    생년월일 입력 시 자동 나이 계산 (펫핀스)
    > 

4️⃣ **경쟁사 분석 인사이트**

<aside>
💡 ‘스마트커버 인슈어런스’는 경쟁사에 비해 **사이트의 정체성이 드러나는 슬로건이 부재**하며 
**보험 정보 입력 및 확인 과정에서 사용자 편의성이 떨어진다**는 문제가 있습니다.

</aside>

# 👥  유저 리서치

> 앞선 문제 인식을 바탕으로 사용자가 ‘스마트커버 인슈어런스’ 사이트를 사용하며 겪을 
페인포인트에 대한 4가지 가설을 설정하고 이를 유저리서치를 통해 검증하였습니다.
> 

<aside>
💭 **[문제 인식을 바탕으로 한 가설 설정]**

1. 사용자들은 카카오톡으로 제공되는 견적서를 파악하는 데에 어려움을 겪을 것이다.
    1. 특히 알아보기 어려운 글씨와 각 보장 별 비교가 어려운 레이아웃이 주된 페인포인트일 것이다.
2. 사용자들은 현재 사이트의 정체성을 파악하는 데에 어려움이 있을 것이다.
    1. 해당 사이트에서 어떤 행동을 할 수 있을 지 파악하기 어려울 것이다.
3. 스마트커버 인슈어런스의 보험 비교 서비스 이용 방법을 파악하는 데에 어려움이 있을 것이다.
4. 견적서 정보 입력 시, 선택지와 예시 미제공으로 불필요한 소요 시간 발생에 불편함을 겪을 것이다.
</aside>

가설을 검증하기 위해 직접 사용자의 목소리를 들어보고자 **유저 리서치를 진행**하였습니다.

## ❇️ 유저 리서치 설계

- 목적: 현재 자사 사이트의 **사용성을 테스트**하고 이를 통해 사용자들의 **페인포인트와 니즈를 파악**하기 위함
- 대상: 현재 스마트커버인슈어런스의 페르소나 타겟인 **2040 여성** (총 5명: 20대 2명, 30대 1명, 40대 2명)
- 일시: 2024. 8. 22. ~ 2024. 8. 25.
- 인터뷰 진행 방식: 대면 or 비대면
    - 비대면의 경우, ZOOM을 이용한 화면 공유를 통해 사용자가 사이트를 활용하는 모습을 실시간으로 확인

## ❇️ 유저 리서치 분석

    “펫보험에 대한 정보가 없이 고객 리뷰가 먼저 나오니 당황스러워요.”
    ”반려생활에서 핏펫몰로 이동한지 몰랐어요. 다시 SC 사이트로 돌아오는 방법을 찾기 어려웠어요.”*

    “한 장에 **너무 많은 정보**가 들어가서 인지가 어려워요. 두 장으로 나누어 왔으면 더 좋을 것 같아요.”*
    >

    - ‘펫보험 견적받기’ 라는 **UX writing이 비교하기 기능을 나타내는 지 파악하지 못하고 헤맴**
    - ‘펫보험 견적받기’ 이후 **견적서를 수령하고 상담받을 수 있는 방법을 명확히 파악하지 못함**
    - 나이를 입력할 때, 생년월일로 기억하니 일일이 나이를 세서 작성함

    ”견적서 내용은 **상세해서 좋아요.** 상담으로 넘어갈 만한 동기가 되는 것 같아요.”

    > 

## ❇️ 가설 검증

일치하지 않았고 보험 사이트에 대한 인지는 되나 기능에 대한 안내가 부족하다는 문제로 정의)**

**⇒ 적합하지 않은 ux writing으로 기능 사용에 대한 어려움을 겪음 (가설 검증 완료 및 채택)**

유저도 빠르게 사이트를 파악할 수 있게 합니다.

또한, 기존 견적서 발송 시 **미흡한 팝업 안내로 인해 사용자가

## ❇️ 페르소나를 고려한 UX

> 현재 스마트커버 인슈어런스의 주요 타겟 페르소나는 비대면을 선호하는 2040 반려인 여성입니다.
해당 페르소나를 고려한 다양한 UIUX를 사이트에 적용하였습니다.
> 

### 1. 반려인을 위한 센서티브한 UI와 UX writing

- 딱딱한 ‘~습니다.’ 체보다 부드럽고 친근한 ~세요.’ 체를 사용하여 2040 반려인 여성에게 거부감 없이 다가가고자 하였습니다.
- 대표 캐릭터인 고양이와 강아지를 사이트 곳곳에 적절하게 배치하여, 사이트에 대한 장벽을 낮추고 브랜드의 긍정적인 이미지를 구축하고자 하였습니다.

### 2. 즉각적인 인지가 가능한 UX

- 완료 후 팝업, 오류 시 메세지, 마우스 호버 시 인터랙션을 통해 해당 사이트의 사용자가 본인의 행동을 빠르게 인지할 수 있도록 하였습니다.
- 이탈 시 팝업을 띄워줌으로써 사용자의 불필요한 행동을 최소화 하였습니다.
- 견적서 입력 시, 사용자 입력에 따라 프로그레스 바가 변하는 인터랙션을 통해 사소한 디테일에서도 사용자의 인지도를 향상하였습니다.

### 3. 보험이 낯설 2040 여성을 위한 충분한 정보 제공

- 보험이라는 낯선 주제를 쉽게 이해할 수 있도록 충분한 정보 제공
    - 메인 화면의 설명
    - 각 기능 별 페이지의 배너
        - 해당 페이지의 기능을 한 줄로 파악할 수 있도록 하였습니다.
    - 보험 용어 설명
        - 어렵고 복잡한 보험 용어를 간단하게 풀어서 제공합니다.
    - 펫보험 팁
        - 기존의 단순한 반려동물 내용에서 보험과 의료에 대한 내용과 연결하여 사이트 내의 내용적 통일성을 높이고 사용자에게 유의미한 정보를 제공합니다.
        - 일부 게시물에서는 보험 견적 받기를 유도하도록 하여 비교 견적서를 받을 수 있는 모수를 확대할 수 있도록 하였습니다.

## 1. 사용성 테스트 설계

사용자에게 특정 페르소나와 상황을 부여하고 이에 따른 과제를 제시하여 해당 사이트를 이용하는 행태와 보완점을 확인하고자 했습니다.

### **페르소나 및 상황**

    - page router

- **TypeScript:** 정적 타입 검사로 코드 안정성을 높이고 디버깅을 쉽게 합니다.
- **TailwindCSS:** 유틸리티 클래스 기반으로 신속하고 일관된 스타일링이 가능합니다.

> **백엔드 스택 선정과 이유**

### **Backend**

- **Layered Architecture in DDD:** 복잡한 비즈니스 로직을 체계적으로 관리하고 확장성을 높입니다.

```
└─fitpet_be
    ├─application
    │  ├─dto
    │  │  ├─request
    │  │  └─response
    │  ├─exception
    │  ├─service
    │  └─serviceImpl
    ├─common
    │  └─config
    ├─domain
    │  ├─model
    │  └─repository
    ├─infrastructure
    │  ├─jwt
    │  ├─persistence
    │  ├─s3
    │  └─utils
    └─presentation
        └─controller
```
- **CQRS:** 읽기와 쓰기 작업을 분리해 성능과 데이터 일관성을 유지합니다.
- **Java 17:** 안정성과 보안을 제공하며, 최신 LTS 기능을 활용할 수 있습니다.
- **Spring Boot:** 빠른 개발과 표준화된 코드 구조를 지원합니다.
- **Spring Data JPA, Swagger, Logback, AWS 등:** 데이터 접근, API 문서화, 로깅, 클라우드 인프라 활용을 간소화합니다.

- **JWT(Jason Web Token)**: 현대 웹 애플리케이션에서 인증과 권한 부여를 위해 널리 사용되는 기술로, 서버와 클라이언트 간의 데이터를 안전하게 주고받을 수 있도록 해줍니다. 관리자 인증, 인가에 사용했습니다.
- **Aspose.Cells**: 보험료 견적서에 사용된 Microsoft Excel를 생성, 읽기, 조작, 변환 및 보호하기 위한 클라우드 API입니다.
- **Apache POI**: Microsoft Excel 포맷을 순수 자바 언어로서 읽고 쓰는 기능을 제공합니다. 보험료 견적서에 사용자의 정보를 입력하기 위해 사용했습니다.
